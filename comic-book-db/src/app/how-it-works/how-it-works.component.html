What Is This Platform?

This is a claim sale platform — a private, invite-only storefront purpose-built for running claim sales online. It handles the full experience: browsing a catalogue, claiming items in real time, cart management, order review, fulfillment, and a permanent sales archive.

The platform is duplicatable for a one-time fee of $3,000. That covers a fully configured, ready-to-run copy of the platform deployed to your own Azure account — your own domain, your own data, completely independent of anyone else's instance.

Once deployed, your ongoing infrastructure costs are approximately $3/month, for Azure usage, the Namecheap DNS + email service. Azure's serverless pricing means you only pay for what you use, and a typical claim sale operation stays well within free-tier limits for most of the month.

The $20/month AI subscription (Claude) is entirely optional. It is only needed during periods when you want to make changes to the site — adding features, adjusting the design, or performing maintenance. When you're not actively developing, you can cancel it and resubscribe only when needed.

What Is a Claim Sale?

A Claim Sale is a first-come, first-served selling format that originated in comic book communities on social media. The seller lists comics at fixed prices and buyers race to claim individual books — no bidding, no auctions, no back-and-forth negotiating. The price is set, and the first person to claim a book gets it.

This site is a private, invite-only storefront built around that format. The seller (that's us) loads the catalogue and sets prices. Approved buyers browse and claim. It's intentionally small and personal — you know exactly who you're buying from, and the seller knows exactly who they're selling to.

Claim sales move fast. If you see something you want, don't wait — someone else might grab it first. Once a book is claimed it's off the table for everyone else until it's released back or the order is cancelled.

You can simulate a "timed claim sale" by putting books in the database without prices. Then make them go live one by one by setting a price — only priced books show a Claim button.

Getting Access

This shop is invite-style — you need an approved account before you can claim anything. Anyone can browse the catalogue, but only approved buyers see the Claim buttons.

  1. Request an account — fill out the registration form with your name, email, shipping address, phone number, and your preferred payment method.
  2. Wait for approval — the seller reviews your request. Once approved, you'll receive a PIN directly from the seller via message or email. This PIN is your password — it's generated once and never stored in plain text.
  3. Log in — head to the login page, enter your email address and PIN, and you're in. No email verification links, no password resets. If you ever lose your PIN, contact the seller to have it reset.

The Dashboard

The Dashboard is the main catalogue — every comic currently available for sale is shown here as a card with its cover image, title, series, grade, and price. Comics that have already been sold are not shown at all.

  • Available comics show a blue Claim button if you're logged in as an approved buyer. Comics without a price set do not show a Claim button — the seller hasn't priced them yet.
  • Claimed comics show a grey Claimed label with the date it was claimed. They're in someone else's active cart — if that order falls through, the book may become available again.
  • Comics in your own cart show a green In Your Cart label so you can see at a glance what you've already claimed.
  • Sets appear as their own card with an orange Set N badge. Clicking a set card takes you to the set's detail page rather than a single-comic detail page.

You can also use the search bar at the top of the dashboard to filter comics by title, series, or other details.

Sets

A Set is a bundle of comics sold together as a single unit. Sets are ideal for runs, lots, or curated groups the seller wants to keep together. You cannot buy individual books out of a set — it's all-or-nothing.

Clicking a set card on the dashboard opens the Set Detail page, which shows:

  • A cover image and description for the set
  • The full list of member books with their individual grades and prices
  • The combined total price for the whole set
  • A Claim Set button (if you're an approved buyer and the set is available)

When you claim a set, every book in the bundle is added to your cart at once, each tagged with the orange Set N badge. If you want to remove the set from your cart, clicking Remove Set on any member book removes the entire set in one action — you cannot remove just one book from a set while keeping the others.

All books in a set must be priced before the Claim Set button appears.

Bidding

Some comics are offered through a bidding format instead of a fixed-price claim. These are easy to spot — they show a Bid button on the dashboard card and detail page rather than a Claim button.

The bidding format used here is a soft-close English auction. That means:

  • Ascending price — each bid must be higher than the current high bid. The highest bidder at the end wins.
  • Open bids — the full bid history (bidder names, amounts, and timestamps) is visible to everyone on the comic's detail page.
  • Inactivity close — there is no fixed end time. The countdown timer resets to the full window every time a new bid is placed. The auction only closes after the entire window passes with no new bids. This prevents last-second sniping — if you get outbid, you always have time to respond.

How to participate

  1. Start the cycle — click Bid on any eligible comic. This opens the bidding cycle at the opening price and makes you the current high bidder. The countdown timer starts immediately and is visible to all buyers.
  2. Place bids — if another buyer outbids you during the window, you'll see the live high bid and their name update on both the dashboard card and the detail page. Enter a higher amount on the detail page and click Place Bid to retake the lead. Each bid resets the timer to the full window.
  3. Winning — when the timer runs out with no new bids, the comic is automatically added to the winner's cart at the final high-bid price. A toast notification announces the result to everyone on the dashboard.

Items won via bidding show a Won via bid tag in your cart and cannot be removed. Contact the seller directly if there is an issue with a bid-won item.

There is no minimum bid increment — any amount strictly above the current high bid is valid.

Comic Detail Pages

Click any comic card on the dashboard to open its detail page. Here you'll find the full information for that book:

  • High-resolution front and back cover images (click to zoom in)
  • Title, series, issue number, publisher, and publication date
  • Grading details — CGC/CBCS certification number and grade, or estimated grade for raw copies
  • Page quality, pedigree, restoration notes, and signatures if applicable
  • Sale price and any public notes from the seller
  • GoCollect and Grand Comics Database links if available

The Claim and Release buttons are also available on the detail page, so you can review a book fully before adding it to your cart, or remove it from your cart directly.

Your Cart

Your cart lives at My Cart in the navigation. Every book you claim goes here. The cart moves through a few stages:

  1. Open — you can freely add and remove comics. There's no time limit while your cart is open.
  2. Submitted (review window) — when you're done shopping, click Submit Order. This starts a short review period set by the seller. During this window you can still remove items but you cannot add new ones. Any applicable sale discounts are calculated and shown at this point.
  3. Finalized — once the review window expires, your cart locks automatically. The seller will reach out to arrange payment and shipping.
  4. Fulfilled — once the seller marks your order as fulfilled, the cart is closed. You can then start a new cart and claim again.

If a discount applies to your order, you'll see a Discount line in your cart summary with the amount saved and a description of which rule was applied. Discounts are automatically recalculated if you remove items from a submitted cart.

Items that belong to a set are grouped and tagged with an orange Set N badge. The Remove Set button removes all books in the set at once — individual set members cannot be removed one at a time.

Orders

The Orders page shows your complete order history — every order that has been fulfilled, with its line items, prices, and totals. This history is permanently archived and will never be lost, even if the seller resets the catalogue between sale events.

Live Notifications

The app shows real-time toast notifications in the upper-right corner of every page, so you always know what's happening even when you're not on the dashboard.

  • When you claim a comic, you'll see a confirmation: "Title #N" added to your cart — $XX.
  • When someone else claims a comic, you'll see: "Title #N" added to UserName's cart — $XX.
  • When the seller awards a comic to a user, everyone sees: "Title #N" awarded to UserName — FREE!
  • When anyone removes a comic from their cart, everyone sees: "Title #N" Returned to sale. This is useful so you know a book you wanted has come back available.

Each notification stays visible for 30 seconds and can be dismissed early with the ✕ button. Multiple toasts can stack. Notifications clear automatically when you navigate to a new page.

Your Profile

Your Profile page lets you update your contact details at any time — name, shipping address, phone number, and payment notes. Keeping these current ensures the seller can reach you and knows how you want to pay when your order is finalized.

Payment

Payment is handled directly between you and the seller — there's no checkout or payment processor built into this site. When you register (or update your profile), include your preferred payment method in the Payment Notes field. Common options include PayPal, Venmo, cash at pickup, and so on. The seller will contact you after your order is finalized to confirm the total, payment, and shipping details.

Contact

The Contact page is the best way to reach the seller outside of a formal order. Use it to:

  • Ask a question about a specific listing
  • Make an offer on something you've seen
  • Inquire about a book not currently in the catalogue
  • Say hello

Fill in your name, email address, and message, then hit Send Message. The seller typically responds within 1–2 business days. You do not need to be logged in to use the contact form.

Admin Reference

Dashboard (Admin View)

When logged in as Admin, the dashboard shows additional controls on each comic card when Award Mode is enabled (toggled from the Sales page).

  • Award button — appears on any unclaimed comic. Clicking it opens a modal listing all approved (non-admin) users. Selecting a user immediately adds the comic to that user's cart at $0.00 (free). If the user has no active cart, one is created for them. Already-claimed comics do not show the Award button.

Bidding (Admin)

Bidding is controlled at two levels: a global on/off switch and a per-comic opt-in.

  • Global switch — set the BIDDING_MODE environment variable to true on the Azure Function App to enable the bidding API. When false (or unset), all bid endpoints are disabled and the dashboard shows normal Claim buttons even on bid-enabled comics.
  • Per-comic opt-in — check the Enable Bid checkbox on a comic row in Comics Management to mark that comic for bidding. Only comics with this flag set will show a Bid button to buyers (when global bidding is also on).
  • Window length — set BIDDING_CYCLE (in minutes, default 10) to control how long the inactivity window is. Each new bid resets the full window. Shorter windows create faster, more urgent cycles; longer windows give buyers more time to respond to an outbid.

No manual action is needed to close a cycle. When the countdown timer on the dashboard hits zero, the browser automatically calls the finalize API, adds the comic to the winner's cart at the high-bid price, and shows a toast notification to all online users. The comic then shows as claimed on the dashboard and is locked out of further bidding.

Bid-won items carry a wonViaBid flag on the cart item. They are priced at the winning bid amount and are excluded from the buyer's ability to remove items, but they are included in discount calculations when the buyer submits their order.

Manage Orders

The Manage Orders screen is split into three sections: Active Orders (submitted and finalized carts), Open Shopping Carts (carts with items that haven't been submitted yet), and Order Archive (permanently fulfilled orders).

Each comic title is a clickable link showing the title and issue number together.

  • Release — removes a single comic from a buyer's cart and returns it to sale. A "Returned to sale" toast is shown to all users in real time. Discounts are automatically recalculated after removal.
  • Mark as Fulfilled — only enabled once a cart reaches Finalized status (after the review window). Uses a two-click confirmation: first click reloads the latest order data, then the button turns orange showing Confirm Fulfill?. The second click permanently archives the order, stamps each comic with the buyer's name and sold date, and removes it from the active list. Fulfilled comics no longer appear on the dashboard.

Carts transition from Submitted to Finalized lazily — the next time the Orders page loads after the review window expires.

The review window length is controlled by the FINALIZE_HOURS environment variable on the Azure Function App. The default is 20 hours if not set. Adjust it via the Azure portal or az functionapp config appsettings set.

The Order Archive section at the bottom shows all permanently fulfilled orders. Each archived order has two delete options:

  • Delete — removes only the archived order record. The comics that were in that order remain in the database (useful if you want to keep the comic records).
  • Full Delete — only appears on orders that contained a set. Removes the archived order record and permanently deletes all comics in the associated set from the database. Use this when you want to fully clean up a sold set.

Order Archive records are never touched by a database reset, so the full sales history is always preserved.

Sales & Discounts

The Sales page lets you configure discount rules automatically applied when a buyer submits their order. Multiple active rules stack. The discount is snapshotted onto the order at submit time, so historical totals are preserved even if you later change or remove a rule.

Three discount types are supported:

  • Flat Percentage Off (RAW_PERCENTAGE) — deducts a fixed percentage from the cart subtotal. Example: 10% off everything.
  • Buy X Get One Free (BUY_X_GET_ONE_FREE) — for every group of X+1 books, the cheapest book in that group is free. Example: Buy 4, get the 5th free.
  • Percentage Per X Books (PERCENTAGE_PER_X_BOOKS) — for every X books in the cart, add a percentage discount (capped at 100%). Example: 5% off per 3 books means a 6-book cart gets 10% off, a 9-book cart gets 15%, and so on.

Each rule can be toggled active or inactive. Inactive rules are saved but not applied. You can create, edit, and delete rules from this screen.

This page also lets you toggle Award Mode — when enabled, an Award button appears on unclaimed comics on the dashboard, letting you add a book to any approved user's cart at $0.00.

Reset Database

At the bottom of the Sales page is a Reset Database section, intended for use between sale events. To confirm the action, type RESET in the input field and click the button. This will:

  • Archive any fulfilled carts that haven't been archived yet (order history is preserved)
  • Delete all active carts and discounts
  • Delete all comics and their associated images
  • Delete all sessions except your own (so you stay logged in)

The Order Archive is never touched by a reset — all past sales history remains intact in the Orders admin view and in each buyer's Orders page.

User Admin

The User Admin screen is split into two tables — pending account requests at the top and existing approved users below.

  • Approve — approves a pending request, generates a PIN, and moves the user to the Approved Users table. A modal displays the new PIN — copy it and send it to the buyer directly. It will not be shown again.
  • Reset PIN — generates a brand-new PIN for an existing user (for example, if they've lost theirs). The same modal appears with the new PIN. The old PIN stops working immediately.
  • Suspend / Reactivate — suspending a user blocks their login immediately without deleting their account. Their status badge turns red. Reactivating restores their access.

The admin account row does not show Reset PIN or Suspend controls — it displays Admin in the Actions column instead.

Comics Management

The Comics Management screen is a full-featured data grid showing every comic in the database — not just the ones listed for sale. Each row is directly editable: click any cell to update it and the change saves automatically.

  • For Sale — the checkbox in this column controls whether a comic appears on the buyer-facing dashboard. Unchecking it hides the comic from buyers immediately without deleting it.
  • Date fields — Date Acquired, Published Date, and Date Sold use a date picker. Enter dates in YYYY-MM-DD format. Setting a Date Sold value on a comic automatically hides it from the dashboard.
  • Sale Price vs Target Price — Target Price is your internal reference; Sale Price is what buyers see on the dashboard and in their cart totals. Books without a Sale Price do not show a Claim button to buyers.
  • Add — use the toolbar button to add a new comic row with placeholder values. Fill in the details directly in the grid.
  • Delete — the delete icon on the left of any row permanently removes the comic, deletes all of its associated images (front cover, back cover, and thumbnails), and automatically releases it from any active carts.
  • Sorting and filtering — click any column header to sort, or use the filter icon to narrow the list by any field.

Sets Management

The toolbar also contains set-specific actions:

  • Add Set — creates a new set container comic. Give it a title, series, cover image, and public notes that describe the bundle. The container itself is priced at $0; the total displayed to buyers is the sum of the member books' sale prices.
  • Add to Set — assigns an existing standalone comic to a set. Select the target set and then the comic to add. The comic becomes a member of that set and will no longer appear as a standalone item on the dashboard.
  • Delete Set — removes the set container and ungroups its member comics, making them standalone again. This is only allowed before the set has been claimed. If any member is currently in a user's cart, or if the set appears in a fulfilled archived order, the delete will be blocked with an explanatory message. To remove a fulfilled set, use Full Delete on the corresponding archived order instead.

Comic Detail Page (Admin View)

When logged in as Admin, comic detail pages include additional image management controls.

  • Update Image — uploads a new front cover image for the comic. The image is automatically resized if it is very large, and a small thumbnail is generated for use on the dashboard. If the upload fails, an error message appears below the button.
  • Update Back Cover — uploads a back cover image. This section only appears after a front cover image has been uploaded. A thumbnail is generated automatically.

Uploaded images are stored with metadata (comic ID, title, series, and issue number) attached to each image record.
